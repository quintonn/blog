Before getting started, it is important to know about GitHub.
If you already know what it is and how it works, feel free to skip ahead to the getting started section. GitHub is basically a web site that allows people to store various kinds of files.
The most important and common types of files stored on GitHub, are source code files.
You can think of these as special kinds of text files that contain the source code to programs and websites. You might think this is just the same thing as DropBox or Google Docs or OneDrive and so on.
The main difference with GitHub is that GitHub also provides a ton of extra features that those others don't, all of which are specific to source code management. These features include keep track of different versions of all the files, controlling who has access to those files and many others.
Another feature that is used to run this and any other Insta Blog, is the ability of GitHub to host websites using the files already stored there.
This does have 1 important requirement, the files are publicly visible and can been seen by anyone. So it is important not to store secrets, passwords, or any valuable or secret information in any of the files inside a public GitHub repository. One more thing to note about GitHub is the concept of repositories. Repositories are basically projects.
This means that all the files in one repository belong to the same project.
You can create as many repositories as you wish. And each repository has it's own set of settings, including if it is public or not.
An example repository is the one that hosts this website you are reading this blog on. It's called InstaBlog. The next important concept to understand is Forking.
Forking is basically the ability to make a copy of someone else's repository, or project, and adding it to your github account.
This also makes it possible to update your version of that repository with any changes that are made in the original repository after you forked it, as well as allow you to provide changed from your repository back to the original repository.
The details of how this is done can be found on GitHub's website. All of this may seem overwhelming, but using Insta Blog require only that you click a few buttons on GitHub's dashboards. This is by no means a comprehensive guide to using GitHub.
For more information, please visit GitHub's website.
To get started with your own blog using Insta Blog you will need a couple of things: A GitHub Account. A very basic knowledge of how GitHub works. The ability to edit text files that look strange (a.k.a. HTML and JSON files).
If this sounds scary or difficult, don't worry, it is not. Just follow along and you'll be fine. Once you have your GitHub account setup, you are ready to get started.

When you are done editing the file, click the commit changes button once again. The next couple of steps will require creating a couple of folders. But creating folders directly in GitHub is not very intuitive, so read the next few sections to the end before attempting to perform each step. Next, you need to create a folder for you category inside the content folder, if it does not already exist.
The name of this folder must be exactly the same as the category in the items.json folder, but all lower case and any spaces need to be replaced with the "_" character.
Then inside your category folder, create a new folder that matches the name of your blog post, again all lower case and replace spaces with "_". And inside this folder there needs to be an index.html file.
To create this file and the required folders, go to the content folder and click "Create new file".
